# Hero 섹션 분석

- 분석 대상: `my-portfolio/src/components/landing/hero-section.jsx`
- 비교 기준: `포트폴리오 기획서.md` (Hero 섹션 컨셉, 나의 포지셔닝)
- 작성일: 2026-07-22

## 1. 헤드라인 임팩트

"Hello, I'm 이지혜"는 평이한 인사말 수준이라 임팩트가 약하다. 기획서가 명시한 "기술중심형" 헤드라인이 실제로는 반영되지 않았다 — 기획서 1번 포지셔닝의 핵심 문장인 "산업디자인 4년 + React/Supabase 실제 구현 역량"이 Hero 어디에도 드러나지 않는다.

## 2. 메시지 명확성

역할 전환 타이핑("웹디자이너입니다" ↔ "풀스택 개발자입니다")은 기획서의 차별화 포인트가 잘 구현돼 있어 누구인지는 전달된다. 다만 왜 이 조합이 강점인지(스타트업·중소기업이 원하는 "혼자 다 되는 신입")까지는 설명이 없어서, 처음 보는 방문자는 단순히 "취미가 여러 개인 사람"으로 읽을 수 있다.

## 3. 시각적 요소

민트 배경색 블록 + 중앙정렬 텍스트뿐이라 다소 밋밋하다. 타이핑 커서 깜빡임 외에는 애니메이션이 전혀 없고, 프로젝트에 이미 정의된 `rise-up` keyframe도 Hero엔 미적용이다. 기획서가 "미니멀 + 애니메이션"을 명시했던 것에 비해 애니메이션 비중이 약하다.

## 4. CTA 버튼

"프로젝트 보기"(primary) / "더 알아보기"(outlined) 구조는 명확하고 우선순위도 적절하다 — 5개 항목 중 가장 잘 되어 있는 부분이다.

## 5. 첫인상에서 드러나는 개발자 역량

기술 스택이 텍스트로도 아이콘으로도 전혀 노출되지 않아, Hero만 보면 "React/Figma/Rhino를 실제로 다루는 사람"이라는 근거가 보이지 않는다. 이 증거는 한참 스크롤해야 나오는 Skill 섹션에만 있다.

## 개선 제안

1. 이름 아래 설명 문구를 기획서 포지셔닝 한 줄 요약에 가깝게 교체 — 예: "산업디자인 4년, 그리고 React·Supabase로 직접 구현까지"
2. 타이핑 문구 아래에 작은 기술 스택 라인/칩 추가 (React · Figma · Illustrator · Rhino) — 스크롤 없이 바로 역량이 보이게
3. Hero 등장 시 `rise-up` 페이드인 애니메이션 적용 — 애니메이션 비중 보강
4. 배경을 단색 대신 `var(--color-primary-light)` → `var(--color-bg-primary)` 은은한 그라디언트로 — 미니멀 유지하면서 밋밋함 완화
5. CTA 버튼 구조는 현재 그대로 유지
