# 포트폴리오 기획서

- 참고 자료: `포트폴리오 벤치마킹 분석.md`(robinwieruch.de, danielsun.space) + 본인 UI/UX 작업 스타일(uxui 뚜레쥬르 리디자인 — 실사 이미지 활용, 시즌 스토리텔링, 정돈된 정보구조, 그린 베이스+포인트컬러 톤)
- 작성일: 2026-07-21
- 대상: `my-portfolio`

## 1. 나의 포지셔닝

- 목표 직무: 웹디자이너, 풀스택 개발자
- 경력 수준: 신입
- 지원 회사 성격: 스타트업, 중소기업
- 나만의 강점: 디자인 감각, 호기심, 넓은 분야
- 포지셔닝 한 줄 요약: **"산업디자인 4년 전공 + 실제 구현(React/Supabase) 역량을 모두 가진, 디자인부터 개발까지 혼자 커버 가능한 신입"** — 한 사람이 여러 역할을 겸해야 하는 스타트업·중소기업 채용 니즈에 정확히 부합.

## 2. Hero 섹션 컨셉

- 참고: robinwieruch.de(신뢰형 텍스트 계층), danielsun.space(임팩트형 초대형 타이포) 두 레퍼런스를 종합
- 헤드라인 스타일: 기술중심형
- 시각적 요소: 미니멀 + 애니메이션
- 색상 톤: 그레이스케일/네이비 베이스 + 민트 포인트컬러 (아래 5번 참고)
- **차별화 포인트: 역할 전환 타이핑 애니메이션**
  - 이름 아래 서브카피가 "웹디자이너입니다" → 지우고 → "풀스택 개발자입니다" 로 타이핑 효과로 반복 전환
  - 신입 포지셔닝에서 "둘 다 할 수 있다"는 메시지를 텍스트 설명이 아니라 애니메이션 자체로 증명하는 방식
  - 구현: 순수 CSS(`@keyframes`) 또는 가벼운 JS 타이핑 로직으로, 별도 라이브러리 추가 없이 미니멀하게 구현 가능

## 3. About Me 섹션 구성

- 개발 시작 스토리: 산업디자인 전공으로 4년을 마친 뒤, 흥미를 느껴 웹디자인 분야로 진입 — "전공을 그대로 잇기보다 흥미를 따라간" 전환의 서사
- 핵심 가치관: 사용자 경험을 최우선으로, 디테일을 놓치지 않으면서도 결과물은 깔끔하게
- 개인적 매력: 네일아트, 사진 찍기 등 오밀조밀한 것을 모으고 꾸미는 걸 좋아하는 맥시멀리스트적 취향 — 그런데 정작 작업물(레이아웃 등)은 미니멀하게 정돈하는 편
  - **스토리텔링 포인트**: "취향은 맥시멀, 결과물은 미니멀"이라는 대비 자체가 개성 — "디테일에 진심인 사람이 오히려 정돈된 결과물을 만든다"는 한 줄로 연결하면 단순 취미 나열보다 훨씬 기억에 남음
- 성장 목표: 유니버설 디자인 지향 — 더 다양한 연령대가 편하게 쓸 수 있는 사용자 경험으로 넓혀가고 싶음
  - 참고: 이미 이 포트폴리오에서 다크모드 카드/배경 대비를 개선한 이력이 있음 — "말로만 하는 목표"가 아니라 실제로 실천한 사례로 자연스럽게 연결 가능

## 4. Skill 섹션 차별화

- 참고: 두 사이트 모두 스킬을 별도 섹션으로 강조하지 않았음(robinwieruch는 역할 카드, danielsun은 서비스 태그) → 오히려 "디자인 툴 + 코드"를 함께 보여주는 스킬 섹션 자체가 차별점이 될 수 있음
- 주력 기술 (5개): **Illustrator, Photoshop, Figma, Rhino, React(코딩)**
  - Rhino(3D 툴)가 포함된 게 핵심 차별점 — 그래픽 툴만 다루는 일반적인 "디자이너 겸 개발자" 포트폴리오와 달리 산업디자인 백그라운드(입체·제품 감각)를 숨기지 않고 드러냄
- 표현 방식: **인터랙티브 카드형** — 카드에 마우스를 올리면 숙련도와 간단한 코멘트(예: "실무 툴", "성장 중") 등이 나타나는 방식. 이미 Projects 섹션에서 쓰고 있는 카드+호버 확대 톤을 그대로 재사용해 컴포넌트를 새로 만들 필요 없이 일관성 유지 가능
- 숙련도: Illustrator 80% / Photoshop 80% / Figma 70% / Rhino 80% / React(코딩) 60%
  - 코딩이 상대적으로 낮은 수치인데, 이건 감점 요소가 아니라 "디자인 베이스가 탄탄한 상태에서 개발 역량을 빠르게 키우고 있다"는 성장 서사로 풀어내는 카피가 함께 필요 (예: 숫자 옆에 "빠르게 성장 중" 같은 짧은 코멘트)

## 5. 전체 디자인 방향

- 전체 컬러 팔레트: **기존 유지** — 민트(#00D2C5) + 네이비(#1B2E5C) + 핑크 포인트(#FF8FB1)
  - "모노톤 + 포인트컬러"라는 원래 아이디어는, 이미 있는 그레이스케일 텍스트(네이비/그레이)를 베이스로 두고 민트를 유일한 포인트컬러로 쓰는 방식으로 재해석 → 전면 재작업 없이 컨셉 충족
- 폰트 스타일: 기존 Roboto 유지 (별도 변경 필요성이 없어 지금 스택 그대로)
- 레이아웃 스타일: **카드형 유지** — Projects 섹션에 이미 구현된 카드 그리드 톤을 Hero/Skill/About까지 확장해 페이지 전체의 일관성을 높임
- 애니메이션 수준: 미니멀~적당함 — 기존 폴더탭 오픈 애니메이션, 방명록 스와이프, 새 Hero 타이핑 효과 정도의 수준을 유지 (화려한 스크롤 트리거 모션은 지양 — danielsun.space 분석에서 확인했듯 과도한 모션은 성능/접근성 트레이드오프가 있음)

## 6. 차별화 전략

- 경쟁자 대비 내 강점: 대부분의 신입 개발자 포트폴리오는 "코드"만, 대부분의 신입 디자이너 포트폴리오는 "비주얼"만 보여줌 — 이 포트폴리오는 산업디자인 4년 + 그래픽 툴(Illustrator/Photoshop/Figma) + 3D 툴(Rhino) + 실제 동작하는 웹앱(React/Supabase)까지 한 사람 안에 있다는 걸 스킬 섹션과 프로젝트 섹션 양쪽에서 증명함
- 타겟 어필 포인트: 스타트업·중소기업은 디자이너와 개발자를 따로 뽑기 어려운 경우가 많음 → "혼자서 기획-디자인-개발까지 커버 가능한 신입"이라는 포지셔닝이 채용 담당자에게 실질적인 리소스 절감으로 읽힘
- 독창적 아이디어:
  1. Hero의 "웹디자이너 ⇄ 풀스택 개발자" 역할 전환 타이핑 애니메이션으로 이중 정체성을 텍스트가 아닌 모션으로 증명
  2. Skill 섹션에 그래픽 툴뿐 아니라 Rhino(3D)까지 넣어 산업디자인 배경을 차별점으로 적극 노출
  3. About Me를 "맥시멀리스트적 취향 vs 미니멀한 결과물"이라는 대비 서사로 구성해 단순 취미 나열이 아닌 기억에 남는 개성으로 전달
  4. "유니버설 디자인 지향"을 선언에 그치지 않고, 이미 실천한 다크모드 대비 개선 작업을 근거로 함께 제시
