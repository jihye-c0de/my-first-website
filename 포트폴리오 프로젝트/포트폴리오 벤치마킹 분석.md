## 1. robinwieruch.de

- 분석 대상: https://www.robinwieruch.de/

### 1-1. 첫인상 분석

- 3초 안의 인상: "개발자 포트폴리오"보다 **1인 컨설팅/교육 비즈니스 랜딩페이지**에 가까움. 이름 → 직함 → 스택 → 위치가 한 줄씩 쌓이고 바로 아래 "Work with Me" 버튼이 있어 "고용 가능한 전문가"라는 신호가 즉각 전달됨.
- 가장 먼저 주목되는 요소: 상단 파란 공지 배너(강좌 홍보) → 큰 볼드체 이름 "Robin Wieruch" → 파란색 서브타이틀 순으로 시선 이동. 사진/일러스트 없이 텍스트와 색으로만 계층을 만듦.
- 톤앤매너: 화려한 비주얼 대신 **숫자와 팩트로 신뢰를 쌓는 절제된 비즈니스 톤** (3,000,000+ 방문자, 80,000+ 학생 등).

### 1-2. 디자인 요소 분석

- 색상 팔레트: 흰색/연회색 배경 + 단일 블루 액센트 + 검정에 가까운 헤딩. 색이 3가지뿐이라 산만하지 않고, 파란색이 링크·버튼·강조 숫자에만 일관되게 쓰여 학습 부담이 적음.
- 타이포그래피: 헤딩은 굵은 산세리프, 본문은 회색 톤 다운. 이름 > 섹션 타이틀 > 본문 순으로 크기 단계가 뚜렷.
- 레이아웃/여백: 중앙 정렬 싱글 컬럼 + 섹션마다 배경색(흰색/연회색)을 번갈아 구분. 별도 구분선 없이도 섹션이 명확히 나뉨.
- 시각적 계층구조: Hero(이름) → 신뢰 지표(숫자) → 서비스(카드) → 상품(강좌) → 사회적 증거(추천사) → CTA(뉴스레터) → Footer. "신뢰 구축 → 제안 → 증명 → 행동 유도" 설득 흐름이 구조에 그대로 녹아있음.

### 1-3. UX 분석

- 네비게이션: Hire Me / Blog / About / Courses 4개뿐. 방문자가 원하는 3가지 행동(고용 문의, 글 읽기, 강좌 구매)에 정확히 대응.
- 정보 구조: "누구인가 → 왜 믿을만한가 → 무엇을 파는가 → 남들이 뭐라 하는가 → 지금 뭘 하면 되는가" 순의 논리적 설득 퍼널.
- 인터랙션/애니메이션: 화려한 모션 거의 없음(정적 카드, 기본 버튼). 콘텐츠 신뢰도에 힘을 준 스타일 — 로딩은 가볍지만 "개성" 요소는 약함.
- 모바일 대응: 3열 카드(서비스, 추천사)가 1열로 자연스럽게 스택, 네비게이션은 햄버거 메뉴로 축소. 데스크톱과 섹션 순서 동일하게 유지되어 일관성 좋음.

### 1-4. 콘텐츠 전략 분석

- 자기소개 방식: "나는 이런 사람이다"보다 **"나와 일하면 이런 결과를 얻는다"**에 초점. 역할을 6가지로 세분화(Technical Lead/Product Engineer/Architect/AI Engineer 등)해 다양한 채용 니즈 대응.
- 프로젝트 소개: **"프로젝트" 섹션이 없고 "강좌(Courses & Books)"가 그 자리를 대체** — 포트폴리오보다 "교육자 겸 컨설턴트" 포지셔닝.
- 연락처/CTA: "Work with Me", "Let's Work Together", 뉴스레터 구독까지 섹션마다 CTA 반복 배치.
- 스토리텔링: 개인 서사 대신 **지표 → 클라이언트 로고 → 업계 리더 추천사(Dan Abramov, Kent C. Dodds 등)**로 신뢰를 "증명"하는 구조.

### 1-5. 장점과 개선점

**강점 3가지**
1. 색상 3가지, 섹션 6~7개로 정보량 대비 군더더기 없음
2. 숫자·유명인 추천사로 "말이 아니라 증거"로 설득
3. 서비스를 역할별로 세분화해 채용 담당자가 원하는 포지션과 바로 매칭 가능

**아쉬운 점**
- 개인적 개성/사진이 없어 "사람"보다 "브랜드"처럼 느껴짐 — 신뢰는 가지만 친근함은 약함
- 실제 "만든 것(프로젝트)"이 안 보이고 강좌 판매 위주라 순수 포트폴리오 목적엔 참고에 한계
- 인터랙션이 거의 없어 실력을 시각적으로 보여주는 요소가 약함

**내 포트폴리오(my-portfolio)에 적용할 아이디어**
- 지금 Hero/About Me 섹션이 플레이스홀더 상태 → Robin처럼 **"이름 → 한 줄 포지셔닝 → 핵심 스택 → CTA 버튼"** 4줄 구조를 Hero에 적용하면 빠르게 완성도를 높일 수 있음
- Projects 섹션(이미 실제 스크린샷 카드 구현됨)에 "개인 프로젝트 2개 · React/Supabase 기반" 같은 숫자 한 줄을 추가해 Robin의 "신뢰 지표" 패턴을 작게 응용

**차별화 방향**: Robin은 "증거 중심의 무개성 신뢰형" → 반대로 **일러스트/색감(민트-네이비 톤)과 방명록 같은 인터랙티브 요소**로 "사람이 느껴지는 포트폴리오"를 미는 현재 방향(폴더탭 애니메이션, 방명록)이 오히려 차별화 포인트.

### 1-6. 타겟 분석

- 목표 직무/회사: 스타트업이 아닌 **중견~대기업의 시니어/리드급 리모트 계약직·컨설팅 포지션**. "US Government, Vodafone, Daikin" 등 클라이언트 로고가 이를 뒷받침.
- 경력 수준: 주니어가 아닌 **시니어~스태프급**. "Technical Lead", "Architect" 역할 제시와 압도적인 팔로워/학생 수는 신입 포트폴리오에서 나올 수 없는 수치.
- 효과성: "숫자와 유명인의 인증"이 실력의 증거가 되는 시니어 채용 시장에는 매우 효과적이지만, 커리어 초반 단계에서 그대로 따라 하면 "증거가 빈약해 보이는" 역효과 가능. 주니어~미들 단계에서는 "신뢰 지표" 전략보다 **"실제 프로젝트를 자세히 보여주는" 전략**이 더 적합.

## 2. danielsun.space

- 분석 대상: https://danielsun.space/
- 참고: 스크롤 트리거 애니메이션 의존도가 높아 정적 스크린샷 도구(microlink)로는 히어로 영역 이후 섹션을 캡처하지 못함. 히어로 스크린샷 + 페이지 텍스트 콘텐츠(WebFetch)로 분석.

### 2-1. 첫인상 분석

- 3초 안의 인상: 오렌지→노랑→흰색 방사형 그라데이션(스포트라이트) 배경 위에 화면 하단까지 꽉 채우는 초대형 볼드 워드마크 "DANIEL SUN"이 압도적인 인상을 줌. 포트폴리오 자체가 "디자인 실력의 증거"로 기능.
- 가장 먼저 주목되는 요소: 방사형 빛줄기 그래픽 → 캐주얼한 인사 문구("Howdy! Meet your trusted design partner,...") → 초대형 이름.
- 톤앤매너: SaaS/Web3라는 다소 딱딱한 타겟군을 상대하면서도 "Howdy!" 같은 캐주얼한 인사와 이모지를 섞어 **자신감 있고 감각적인 크리에이티브 디렉터** 인상을 줌.

### 2-2. 디자인 요소 분석

- 색상 팔레트: 오렌지-옐로우-화이트 방사형 그라데이션 + 검정 텍스트. 화려해 보이지만 실제 색상은 2~3톤으로 절제됨(스포트라이트 하나의 그래픽 요소가 화려함을 담당).
- 타이포그래피: 극단적으로 큰 디스플레이 산세리프(이름) vs 작고 옅은 회색 본문 — 크기 대비만으로 강력한 계층 구조를 만듦.
- 레이아웃/여백: 상단 고정 캡슐형(pill) 네비게이션이 어두운 배경색으로 항상 도드라짐(페이지 배경이 밝아도 대비 유지). 배경 그래픽 위에 텍스트를 겹치는 레이어드 구성.
- 시각적 계층구조: 배경 그래픽 → 인사 카피 → 초대형 이름 로고 순. Work/Story/Process 섹션은 스크롤에 따라 순차 등장하는 것으로 추정(모션 중심 스토리텔링).

### 2-3. UX 분석

- 네비게이션: Work / Story / Process / Connect + "Start project" CTA. 회사소개형이 아니라 **에이전시·프리랜서 영업 퍼널**에 가까운 구성(포트폴리오 → 자기소개 → 작업방식 → 컨택).
- 정보 구조: "무엇을 하는지 → 어떤 사람인지 → 어떻게 일하는지 → 어떻게 연락하는지" 순으로, Robin과 유사하게 논리적 설득 흐름을 따름.
- 인터랙션/애니메이션: 스크롤 트리거 모션이 매우 적극적으로 사용된 것으로 추정(정적 캡처 도구로 히어로 이후 구간을 담아내지 못할 정도). 강한 "감각 있는 디자이너"라는 인상을 주지만 로딩/성능, 모션 민감 사용자 접근성 부담 가능.
- 모바일 대응: 직접 캡처는 실패했으나, 큰 타이포+여백 중심 디자인은 일반적으로 좁은 화면에서도 반응형 대응이 수월한 편.

### 2-4. 콘텐츠 전략 분석

- 자기소개 방식: "우크라이나에서 자란 거리 미술 → 건축학 → 포르투갈 정착"이라는 개인 서사를 스토리텔링으로 전개 — Robin과 정반대로 "사람 냄새 나는" 브랜드를 구축.
- 프로젝트 소개: Ruby, Stimulate, Caldera, Pix AI 등 12개 프로젝트를 나열하되 각 항목에 "브랜딩/웹 디자인/개발" 같은 서비스 태그 표기. "새로운 사례는 천천히 하나씩 준비 중 😅"이라는 자조적 유머로 미완성 부분도 개성으로 승화.
- 연락처/CTA: "Connect" 버튼 반복 + 이메일(hello@danielsun.space) 직접 노출 + 트위터·인스타그램·드리블·링크드인 소셜 링크.
- 스토리텔링: 협업 철학을 "성숙한 정체성으로 사용자 참여와 투자자 신뢰를 높인다"는 비즈니스 언어로 마무리 — 감성적 개인 서사와 비즈니스 임팩트를 함께 제시.

### 2-5. 장점과 개선점

**강점 3가지**
1. 초대형 타이포+빛 그래픽으로 "브랜드 디자이너"라는 직무 자체를 시각적으로 증명(포트폴리오 자체가 작업물의 데모)
2. 개인 서사(우크라이나 → 건축학 → 포르투갈)로 Robin과 달리 강한 개성과 기억에 남는 스토리 구축
3. 유머러스한 문구("Howdy", 이모지)로 SaaS/Web3라는 딱딱한 타겟군에도 친근하게 접근

**아쉬운 점**
- 헤비한 스크롤 애니메이션 의존도가 높아 보여 로딩/성능, 접근성(모션 저감 설정 사용자) 리스크가 있음
- JS 의존도가 매우 높아(정적 캡처 도구조차 렌더링에 실패) 검색엔진 등 크롤러 접근성(SEO)에 불리할 가능성
- 프로젝트 목록에 "천천히 준비 중"이라는 문구가 있어, 최신성 관리가 안 되면 오히려 신뢰도에 마이너스가 될 수 있음

**내 포트폴리오(my-portfolio)에 적용할 아이디어**
- Hero에 이름을 초대형 타이포로 배치하는 기법을 우리 톤앤매너(민트-네이비)에 맞게 미니멀한 버전으로 응용하면 현재 약한 Hero의 시각적 임팩트를 보완할 수 있음
- About Me에 실제 서사(전공/전환 계기 등)를 스토리텔링으로 넣으면, Robin의 "지표 중심" 방식보다 우리 방명록·폴더탭 같은 "사람 냄새" 컨셉과 훨씬 잘 어울림

**차별화 방향**: 이 사이트는 "모션·타이포 강도"로 임팩트를 주지만 과도한 모션은 성능·접근성 트레이드오프가 있음. 우리는 이미 갖춘 적당한 인터랙션(폴더탭, 방명록) 수준을 유지하면서, 실제 프로젝트 스크린샷·라이브 데모 링크 같은 "신뢰 요소"에 집중하는 편이 리소스 대비 효율적.

### 2-6. 타겟 분석

- 목표 직무/회사: "Y-Combinator 창업자들이 신뢰하는" 문구에서 보듯, **초기 스타트업 창업자·SaaS/Web3 파운더 대상 프리랜서 브랜드·웹 디자이너** 포지셔닝. 채용이 아니라 프로젝트 수주가 목적.
- 경력 수준: 이름 단독 브랜드 + 12개 이상의 클라이언트 프로젝트 레퍼런스 — **경력 있는 프리랜서/1인 에이전시 디자이너**(주니어 아님).
- 효과성: 스타트업 창업자는 "감각"과 "즉각적 임팩트"에 반응하는 경우가 많아 초대형 타이포+모션 전략이 효과적. 다만 기업 채용 담당자를 타겟으로 하는 개발자 포트폴리오라면 이 정도의 화려함보다 Robin처럼 "신뢰성/명확성"을 우선하는 편이 더 적합할 수 있음.
