== SNS 사이트 DB 구조 정리 (ate 기획서 기준) ==

사용자 정보 (ate_users 테이블)
- 로그인/회원가입, 프로필, 마이페이지 화면에서 필요한 정보들:
  * 사용자 번호: (자동으로 1, 2, 3... 증가)
  * 사용자명: (@ 표시되는 고유 아이디)
  * 이메일: (이메일 로그인할 때 사용)
  * 비밀번호: (이메일 로그인할 때 사용, SNS 로그인 시에는 비워둘 수 있음)
  * 표시 이름: (프로필에 보이는 실제 이름)
  * 소개글: (프로필에 보이는 자기소개)
  * 프로필 사진: (프로필 이미지 경로)
  * 가입일: (언제 가입했는지)

게시물 정보 (ate_posts 테이블)
- 메인 피드, 게시물 목록(탐색), 게시물 상세, 마이페이지 캘린더 화면에서 필요한 정보들:
  * id: (게시물 고유번호 - 자동으로 1, 2, 3... 증가)
  * user_id: (작성자 ID - ate_users 테이블과 연결)
  * caption: (사진에 쓴 글 내용)
  * image_url: (촬영 후 배경 제거(누끼) 처리된 음료/음식 사진 경로)
  * category: (카페 / 음식점 - 게시물 목록 페이지 카테고리 필터용)
  * place_name: (방문한 카페·음식점 이름)
  * place_url: (네이버 지도 링크 - 게시물 상세, 지도 페이지에서 사용)
  * likes_count: (좋아요 수 - 현재 좋아요 개수를 숫자로 저장)
  * created_at: (작성일 - 언제 올렸는지, 마이페이지 캘린더에서 날짜별로 묶어서 보여줄 때 사용)

게시물 작성 방식:
- 1단계: 하단 탭 중앙 +버튼 클릭 → 카메라 실행 → 음료/음식 촬영
- 2단계: 촬영한 사진 자동으로 배경 제거(누끼) 처리 → 스티커 형태 미리보기 제공
- 3단계: 글귀(caption) 작성
- 4단계: 카페/음식점 카테고리 선택 + 장소 태그 입력 (네이버 지도 링크 연결)
- 5단계: 처리된 이미지 경로와 입력 정보를 ate_posts 테이블에 저장

댓글 정보 (ate_comments 테이블)
- 게시물 댓글을 위한 정보들:
  * 댓글 번호: (자동으로 1, 2, 3... 증가)
  * 댓글 내용: (댓글에 쓴 글)
  * 댓글 작성자: (누가 댓글을 썼는지 - ate_users 테이블과 연결)
  * 댓글 달린 게시물: (어떤 게시물에 댓글을 달았는지 - ate_posts 테이블과 연결)
  * 댓글 작성일: (언제 댓글을 썼는지)

팔로우 정보 (ate_follows 테이블)
- 팔로우/언팔로우 시스템, 메인 피드(팔로우한 유저 글만 노출)에 필요한 정보들:
  * follower_id: (팔로우를 하는 사용자 - ate_users 테이블과 연결)
  * following_id: (팔로우를 받는 사용자 - ate_users 테이블과 연결)
  * created_at: (팔로우한 날짜)

저장 정보 (ate_saved_posts 테이블)
- 피드 저장, 저장한 피드 컬렉션 기능에 필요한 정보들:
  * user_id: (저장한 사용자 - ate_users 테이블과 연결)
  * post_id: (저장된 게시물 - ate_posts 테이블과 연결)
  * created_at: (저장한 날짜)

SNS 테이블 연결 관계 (단순화 버전)
- 사용자 게시물: 한 사용자가 여러 게시물 작성 가능 (촬영 → 누끼 처리 → 카테고리/장소 태그 → 저장)
- 게시물 내 좋아요: 각 게시물마다 likes_count 필드로 좋아요 수 관리 (별도 테이블 없음)
- 사용자 팔로우: 한 사용자가 여러 사용자를 팔로우 가능, ate_follows 테이블로 관계 관리
- 사용자 댓글: 한 사용자가 여러 게시물에 댓글 가능
- 게시물 댓글: 한 게시물에 여러 사용자가 댓글 가능
- 사용자 저장: 한 사용자가 여러 게시물을 저장 가능, ate_saved_posts 테이블로 관계 관리
- 마이페이지 캘린더: ate_posts 테이블의 user_id + created_at 기준으로 날짜별 그룹핑해서 조회 (별도 테이블 없음)
