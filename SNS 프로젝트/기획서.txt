=====================================
모바일 서비스 UI 기획서
=====================================

1. 서비스 기본 정보
-------------------------------------
서비스 이름: ate
서비스 주제: 방문한 카페나 음식점을 자랑하며 기록
핵심 가치: 방문한 카페/음식점 기록이 자동으로 "먹은 것 캘린더(다이어리)"로 쌓여, SNS 피드로 자랑하는 동시에 나만의 미식 기록장이 되는 서비스. 사진 속 음료·음식이 자동으로 누끼 따져 스티커처럼 남아 다이어리 감성을 살림
타겟 사용자: 카페 투어·맛집 탐방을 즐기고 그 기록을 감성적으로 남기고 공유하고 싶은 20~30대. 다이어리 꾸미기(다꾸)나 미니멀한 감성 콘텐츠를 좋아하는 사용자

2. 디자인 컨셉
-------------------------------------
메인 색상: 종이질감의 따뜻한 화이트
보조 색상: 검은색 회색 베이지톤(아이보리톤)
전체 분위기: 깔끔하고 따뜻한 다이어리같은 느낌
참고: SNS 프로젝트\참고 이미지 , 홈이나 탐색/알림은 인스타그램 참고

3. 필요한 페이지 목록
-------------------------------------
[ ] 1. 로그인/회원가입 페이지
    - 기능: 이메일 로그인, SNS 로그인 등
    - 특징: 종이질감 아이보리 배경 위에 심플한 로고와 문구만 배치, 온보딩 최소화

[ ] 2. 메인 페이지 (홈 화면)
    - 기능: 검색 탭이랑 피드 구경
    - 특징: 인스타그램 홈 참고 - 팔로우한 유저의 카드형 피드(사진+캡션+좋아요/댓글 수)를 세로 스크롤로 노출, 배경은 따뜻한 화이트 톤

[ ] 3. 게시물 목록 페이지
    - 기능: 카페/음식점 카테고리별 탐색, 정사각형 그리드로 게시물 모아보기, 인기순/최신순 정렬
    - 특징: 인스타그램 탐색 탭 참고 - 촘촘한 정사각형 그리드, 썸네일은 배경 제거된 음료/음식 사진이 도드라지도록 여백 있는 카드 형태

[ ] 4. 게시물 상세 페이지
    - 기능: 사진 확대보기, 좋아요/댓글/저장/공유, 방문 장소 네이버 지도 링크 연결, 작성자 프로필 이동
    - 특징: 참고 이미지(저기요 포스터)처럼 음식 사진을 여백 있게 배치, 캡션과 위치 태그는 사진 아래 담백하게 배치

[ ] 5. 게시물 작성 페이지
    - 기능: 하단 탭 가운데에 위치한 +버튼을 누르면 카메라가 켜지고 그 후에 글귀를 작성
    - 특징: 자동으로 음료컵이나 접시 누끼 따줌 - 촬영 즉시 배경이 제거되어 스티커처럼 붙는 미리보기 제공, 장소 태그 입력란 포함

[ ] 6. 마이페이지
    - 기능: 프로필 정보(닉네임/소개/프로필 사진), 팔로워·팔로잉 목록, 내가 쓴 게시물 모아보기, "먹은 기록 캘린더" 조회
    - 특징: 참고 이미지(캘린더형 다이어리)처럼 월별 달력 각 날짜 칸에 그날 올린 음료/음식의 누끼 딴 사진을 작게 표시, 날짜 클릭 시 해당 게시물 상세로 이동, 종이질감 아이보리 배경 + 손글씨 느낌의 포인트 문구로 다이어리 감성 강조

[ ] 7. 추가 페이지 (필요시)
    - 페이지명: 지도 페이지
    - 기능: 내가 방문한 카페/음식점을 네이버 지도 위에 핀으로 모아보기
    - 특징: 부가 기능(네이버 지도 링크 연결)과 연동, 우선순위는 3차 개발로 후순위 배치

4. 핵심 기능 정의
-------------------------------------
필수 기능 (꼭 있어야 하는 기능)
1. 프로필 생성 및 관리
2. 사진 게시물 작성, 수정, 삭제
3. 팔로우/ 언팔로우 시스템
4. 피드 좋아요, 댓글, 저장

부가 기능 (있으면 좋은 기능)
1. 네이버 지도 링크 연결
2. 공유하기
3. 1:1 다이렉트 메세지
4. 저장한 피드 컬렉션
5. 사진 자동으로 배경 없애기 (누끼)

5. 모바일 UI 특징
-------------------------------------
네비게이션: 
    - 하단 탭
    - 중앙에 카메라 연결되는 버튼 (촬영 후 바로 게시글 작성 되게끔)
    - 구성 : 홈/ 탐색/ +작성 / 알림/ 마이페이지(캘린더 구성)
주요 버튼 위치: 
    - FAB : 하단 탭 중앙
    - 좋아요 버튼 : 피드 하단 좌측
    - 댓글 작성창 : 좋아요 버튼 옆 (아이콘으로 있다가 클릭하면 댓글창과 함께 작성창 나오는)
터치 영역 크기: 
    - 터치 가능한 요소의 최소 크기 : 44*44px
    - 버튼 간 간격 : 8px
세로/가로 모드: 세로모드만

6. 개발 우선순위
-------------------------------------
1차 개발: 로그인/회원가입, 메인 페이지(피드), 게시물 작성(사진 업로드 + 글귀, 누끼 제외), 게시물 상세, 게시물 목록(탐색)

2차 개발: 팔로우/언팔로우, 좋아요·댓글·저장, 마이페이지 기본 프로필 + 캘린더 뷰

3차 개발: 사진 자동 배경 제거(누끼), 네이버 지도 연동, 지도 페이지, 1:1 다이렉트 메세지, 저장한 피드 컬렉션, 공유하기
