== 최애소(mfi) 커뮤니티 사이트 DB 구조 정리 ==
(mfi = my favorite introduce, 사이트명 "최애를 소개합니다"의 약자)

사용자 정보 (mfi_users 테이블)
- 회원가입 화면에서 필요한 정보들:
  * 사용자 번호: (자동으로 1, 2, 3... 증가) - PK
  * 이름:
  * 이메일: (로그인할 때 사용, 중복 불가)
  * 비밀번호:
  * 전화번호:
  * 가입일: (언제 가입했는지)
  * 회원 상태: (정상 / 정지 - 관리자 제재 반영, 기본값 정상)
  * 제재 사유 및 만료일: (제재된 경우에만 값 존재)

작품 정보 (mfi_works 테이블)
- 소설/웹툰 작품 목록/상세 화면에서 필요한 정보들 (관리자가 등록/관리):
  * 작품 번호: (자동으로 1, 2, 3... 증가) - PK
  * 작품명:
  * 작품 유형: (소설 / 웹툰)
  * 작가:
  * 표지 이미지: (관리자만 등록/수정 가능, 사용자 업로드 금지)
  * 원작 플랫폼 링크: (교보문고, 카카오페이지, 네이버 시리즈 등)
  * 등록일:

태그 사전 (mfi_tags 테이블)
- 관리자가 지정하는 정형화된 태그 목록:
  * 태그 번호: (자동으로 1, 2, 3... 증가) - PK
  * 태그명: (예: 로맨스판타지, 회귀, 힐링)
  * 등록일:

작품-태그 연결 (mfi_work_tags 테이블)
- 작품 하나에 여러 태그, 태그 하나에 여러 작품이 연결되는 관계 테이블:
  * 연결 번호: (자동으로 1, 2, 3... 증가) - PK
  * 작품: (어떤 작품인지 - mfi_works 테이블과 연결)
  * 태그: (어떤 태그인지 - mfi_tags 테이블과 연결)

게시물 정보 (mfi_posts 테이블)
- 독후감/리뷰 게시물 목록/상세 화면에서 필요한 정보들:
  * 게시물 번호: (자동으로 1, 2, 3... 증가) - PK
  * 제목:
  * 내용:
  * 작성자: (어떤 사용자가 썼는지 - mfi_users 테이블과 연결)
  * 작품: (어떤 작품에 대한 게시물인지 - mfi_works 테이블과 연결)
  * 작성일: (언제 썼는지)
  * 수정일: (마지막으로 언제 수정했는지)

게시물 첨부 사진 (mfi_post_photos 테이블)
- 독후감/리뷰용 개인 사진(팬아트, 메모 캡처 등) 업로드 정보:
  * 사진 번호: (자동으로 1, 2, 3... 증가) - PK
  * 게시물: (어떤 게시물에 첨부된 사진인지 - mfi_posts 테이블과 연결)
  * 이미지 경로:
  * 업로드일:

댓글 정보 (mfi_comments 테이블)
- 댓글 화면에서 필요한 정보들:
  * 댓글 번호: (자동으로 1, 2, 3... 증가) - PK
  * 댓글 내용:
  * 작성자: (어떤 사용자가 썼는지 - mfi_users 테이블과 연결)
  * 게시물: (어떤 게시물에 달린 댓글인지 - mfi_posts 테이블과 연결)
  * 부모 댓글: (답글인 경우 원 댓글 번호 - mfi_comments 테이블 자기참조, 답글 아니면 빈 값)
  * 스포일러 여부: (블라인드 처리 대상인지 true/false)
  * 신고 여부: (신고 누적 시 true)
  * 작성일: (언제 썼는지)

별점 정보 (mfi_ratings 테이블)
- 작품별 별점 등록 정보 (0.5점 단위):
  * 별점 번호: (자동으로 1, 2, 3... 증가) - PK
  * 사용자: (누가 매겼는지 - mfi_users 테이블과 연결)
  * 작품: (어떤 작품에 매긴 별점인지 - mfi_works 테이블과 연결)
  * 점수: (0.5 ~ 5.0, 0.5 단위)
  * 등록일:
  * (한 사용자는 같은 작품에 별점 1개만 등록 가능)

좋아요 정보 (mfi_likes 테이블)
- 게시물/댓글 공통 좋아요 정보:
  * 좋아요 번호: (자동으로 1, 2, 3... 증가) - PK
  * 사용자: (누가 눌렀는지 - mfi_users 테이블과 연결)
  * 대상 종류: (게시물 / 댓글)
  * 대상 번호: (mfi_posts 또는 mfi_comments 테이블의 번호)
  * 등록일:

책갈피/메모 정보 (mfi_bookmarks 테이블)
- 인덱스 기능: 회차 단위 책갈피 및 나만의 메모:
  * 책갈피 번호: (자동으로 1, 2, 3... 증가) - PK
  * 사용자: (누가 등록했는지 - mfi_users 테이블과 연결)
  * 작품: (어떤 작품인지 - mfi_works 테이블과 연결)
  * 회차: (몇 화/몇 권 지점인지)
  * 메모 내용:
  * 등록일:

나의 책장 정보 (mfi_bookshelf 테이블)
- 마이페이지 내 독후감/읽기 중단 작품 관리:
  * 책장 번호: (자동으로 1, 2, 3... 증가) - PK
  * 사용자: (누구의 책장인지 - mfi_users 테이블과 연결)
  * 작품: (어떤 작품인지 - mfi_works 테이블과 연결)
  * 상태: (완독 / 읽는 중 / 읽기 중단)
  * 마지막 업데이트일:

신고 정보 (mfi_reports 테이블)
- 게시물/댓글 신고 처리 (관리자 기능):
  * 신고 번호: (자동으로 1, 2, 3... 증가) - PK
  * 신고자: (누가 신고했는지 - mfi_users 테이블과 연결)
  * 대상 종류: (게시물 / 댓글)
  * 대상 번호: (mfi_posts 또는 mfi_comments 테이블의 번호)
  * 신고 사유: (원문 복제, 스포일러 미표시, 욕설 등)
  * 처리 상태: (접수 / 처리 중 / 처리 완료)
  * 신고일:

테이블 연결 관계
- 사용자 → 게시물: 한 사용자가 여러 게시물 작성 가능
- 사용자 → 댓글: 한 사용자가 여러 댓글 작성 가능
- 사용자 → 별점: 한 사용자가 여러 작품에 별점 등록 가능 (작품당 1개)
- 사용자 → 좋아요: 한 사용자가 여러 게시물/댓글에 좋아요 가능
- 사용자 → 책갈피: 한 사용자가 여러 작품에 여러 책갈피/메모 등록 가능
- 사용자 → 책장: 한 사용자가 여러 작품을 책장에 등록 가능
- 사용자 → 신고: 한 사용자가 여러 건 신고 가능
- 작품 → 게시물: 한 작품에 여러 게시물(독후감) 작성 가능
- 작품 → 별점: 한 작품에 여러 사용자의 별점 등록 가능
- 작품 → 태그: 한 작품에 여러 태그, 한 태그에 여러 작품 연결 가능 (다대다, mfi_work_tags로 연결)
- 게시물 → 댓글: 한 게시물에 여러 댓글 작성 가능
- 게시물 → 사진: 한 게시물에 여러 사진 첨부 가능
- 댓글 → 댓글(답글): 한 댓글에 여러 답글 작성 가능 (자기참조)

참고: 마이페이지 취향 그래프는 별도 테이블 없이 mfi_ratings, mfi_likes, mfi_bookshelf 데이터를 작품-태그(mfi_work_tags) 기준으로 집계해서 계산.
